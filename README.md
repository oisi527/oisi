# PT 스케줄러

퍼스널 트레이너 1인을 위한 회원 관리 + 주간 PT 시간표 자동 스케줄링 웹앱.

- **월~일 / 24시간제** 주간 시간표
- **회원 등록** — 이름, 연락처, 주당 희망 횟수, 색상, 메모
- **회원별 가능 시간 등록** — 드래그로 칠하는 7일 × 24시간 그리드
- **자동 배정** — 등록된 가능 시간과 운영 시간을 토대로 최적 시간표 계산
- **수동 수정** — 칸을 클릭해 직접 배정하거나, 세션을 드래그해서 이동
- **고정(🔒)** — 고정한 세션은 자동 배정을 다시 돌려도 유지
- **반응형** — PC / 태블릿 / 모바일, 다크 모드, 인쇄(PDF) 지원

빌드 도구도 서버도 필요 없는 순수 정적 사이트입니다. `index.html`을 열면 바로 동작합니다.

---

## 자동 배정은 어떻게 동작하나

배정 문제를 **최소비용 최대유량(min-cost max-flow)** 으로 풉니다.

```
출발 ──(주당 횟수)──▶ 회원 ──(하루 제한)──▶ 회원·요일 ──▶ 슬롯 ──(1명)──▶ 도착
```

- **최대유량**이므로 배정 가능한 세션 수의 **최대치가 수학적으로 보장**됩니다.
  순서대로 채우는 방식(탐욕)은 "먼저 온 회원이 좁은 회원의 유일한 자리를 차지해버리는" 상황을
  놓치지만, 이 방식은 놓치지 않습니다.
- **최소비용**이므로 그 안에서 분산 배정과 경합 회피가 가장 좋은 조합이 선택됩니다.
- 회원 25명 × 주 3회(75세션) 기준 **10~20ms**에 계산됩니다.

다 채우지 못한 회원이 있으면 **이유와 함께** 알려줍니다
(가능 시간 부족 / 요일 부족 / 다른 회원과 경합).

### 배정 규칙 (설정 탭에서 변경)

| 규칙 | 설명 |
|---|---|
| 하루 최대 1회 | 같은 회원을 하루에 두 번 이상 넣지 않음 |
| 고르게 분산 | 한 회원의 세션을 여러 요일에 흩어 배치 |

## 사용 순서

1. **설정** 탭 → 요일별 운영 시간을 지정합니다 (기본: 월~토 06:00~22:00, 일요일 휴무).
2. **회원** 탭 → `+ 회원 등록` 으로 회원과 주당 횟수를 넣습니다.
3. 등록 직후 뜨는 **가능 시간** 그리드에서 그 회원이 올 수 있는 시간을 드래그로 칠합니다.
   - 다른 회원과 시간대가 같다면 `다른 회원에서 복사`가 편합니다.
4. **시간표** 탭 → `자동 배정` 을 누릅니다.
5. 마음에 안 드는 칸은 클릭해서 바꾸거나 드래그로 옮깁니다.
   - 확정된 세션은 🔒 로 **고정**해 두면 다음 자동 배정에도 그대로 남습니다.
6. `인쇄 / PDF` 로 주간 시간표를 뽑을 수 있습니다.

## 데이터 저장

모든 자료는 **브라우저의 localStorage**에만 저장됩니다. 서버로 전송되지 않습니다.

> ⚠️ 이 말은 곧 **기기·브라우저마다 데이터가 따로**라는 뜻입니다.
> 휴대폰과 PC의 시간표는 공유되지 않으며, 브라우저 데이터를 지우면 함께 사라집니다.
> **설정 → 내보내기**로 주기적으로 백업하고, 다른 기기에서는 **불러오기**로 옮기세요.

시크릿(사생활 보호) 모드에서는 창을 닫는 순간 데이터가 사라지니 주의하세요.

## 배포

정적 파일이라 어디에 올려도 동작합니다.

### GitHub Pages

저장소 → **Settings → Pages → Source: Deploy from a branch → `main` / `(root)`** → Save.
1분쯤 뒤 `https://<사용자명>.github.io/<저장소명>/` 에서 열립니다.

### 일반 웹서버 (Apache / Nginx / 호스팅 FTP)

폴더 전체를 문서 루트에 그대로 올리면 끝입니다. 별도 설정이 필요 없습니다.

### 로컬에서 그냥 쓰기

`index.html` 더블클릭. `file://` 로도 모든 기능이 동작합니다.

## 파일 구조

```
pt/
├── index.html              화면 구조
├── css/
│   └── style.css           스타일 · 반응형 · 다크모드 · 인쇄
└── js/
    ├── util.js             공통 헬퍼 ($, esc, toast, readableOn)
    ├── store.js            데이터 모델 + localStorage (+ 손상 데이터 방어)
    ├── scheduler.js        자동 배정 엔진 (min-cost max-flow)
    ├── ui-members.js       회원 등록 / 수정 / 삭제
    ├── ui-availability.js  가능 시간 그리드 (드래그 페인팅)
    ├── ui-timetable.js     시간표 · 수동 편집 · 드래그 이동
    ├── ui-settings.js      운영 시간 · 규칙 · 백업
    └── app.js              탭 전환 · 초기화
```

의존성 없음. 빌드 없음. 프레임워크 없음.
